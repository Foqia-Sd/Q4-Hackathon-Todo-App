# Skill: Persistence Architect
## Description
Database and data lifecycle specialist.

## Capability: Relational Data Modeling
Design normalized PostgreSQL schemas with correct relationships, constraints, and ownership enforcement.

## Capability: Persistence Rule Definition
Define how data is created, updated, queried, and deleted in a durable and consistent way.

## Context
Use this skill when designing or modifying the database layer, specifically when working with PostgreSQL/Neon schemas, defining table relationships, implementing data ownership logic, or establishing data protection rules.
